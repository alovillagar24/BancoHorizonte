Referencias y sustento técnico
-> Este documento detalla las fuentes académicas y técnicas que fundamentan la implementación de las estructuras de datos y la lógica de negocio del sistema.

1. Sustento Algorítmico
Para la implementación de los gestores de datos (Cola, Árboles, Grafos, etc.), se han seguido los principios descritos en la literatura fundamental de algoritmos:
-> Algoritmos y Estructuras de Datos:
---> Cormen, T. H., Leiserson, C. E., Rivest, R. L., & Stein, C. (2022). Introduction to algorithms (4th ed.). MIT Press. https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/
---> Sedgewick, R., & Wayne, K. (2011). Algorithms (4th ed.). Addison-Wesley Professional. https://algs4.cs.princeton.edu/home/

2. Especificación Técnica (Lenguaje)
La lógica del sistema ha sido construida siguiendo los estándares definidos para el ecosistema de Java, asegurando la compatibilidad y el rendimiento de las clases implementadas:
-> API Documentation:
---> Oracle. (2026). Java Platform, Standard Edition & Java Development Kit Version 21 API Specification. Oracle Corporation. https://docs.oracle.com/en/java/javase/21/docs/api/index.html

3. Metodología de Diseño
Para la creación de este software, se adoptaron prácticas de:
---> Diseño Modular: Basado en la separación de responsabilidades, donde cada clase Gestor encapsula una estructura específica, facilitando el mantenimiento y la escalabilidad del sistema.
---> Modelado UML: Utilizado para la validación lógica de las interacciones entre los componentes del sistema antes de su codificación.
